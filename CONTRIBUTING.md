# Contributing to CNSP Study Guide

Thank you for your interest in contributing to this CNSP study guide! This is a community-driven project, and we welcome contributions from everyone.

## How to Contribute

### 1. Reporting Issues
If you find errors, outdated information, or have suggestions:
- Open an issue on GitHub
- Provide clear description of the problem
- Include relevant details (page, section, etc.)

### 2. Adding Practice Questions
We're always looking for more practice questions:
- Ensure questions are relevant to CNSP syllabus
- Provide correct answers with explanations
- Follow the existing format in `docs/practice-questions.md`
- Don't include actual exam questions (only similar practice questions)

### 3. Improving Documentation
Help improve the study materials:
- Fix typos and grammatical errors
- Clarify confusing explanations
- Add more examples or diagrams
- Update outdated information

### 4. Sharing Study Resources
Know of great study resources?
- Add them to `docs/study-materials.md`
- Include brief description of why it's helpful
- Ensure resources are freely accessible or widely available

### 5. Sharing Your Experience
Passed the CNSP? Share your story:
- Add your tips and insights
- Help others learn from your experience
- Be honest about what worked and what didn't

## Contribution Guidelines

### Content Standards
- **Accuracy**: Ensure all information is accurate and up-to-date
- **Clarity**: Write in clear, understandable language
- **Relevance**: Keep content focused on CNSP exam preparation
- **Respect**: Don't share actual exam questions or violate exam policies

### Formatting
- Use Markdown formatting consistently
- Follow existing document structure
- Use proper headings and sections
- Include tables where appropriate for better readability

### Code of Conduct
- Be respectful and professional
- Welcome newcomers and help them learn
- Provide constructive feedback
- Focus on the content, not the person

## Pull Request Process

1. Fork the repository
2. Create a new branch for your changes
3. Make your changes following the guidelines above
4. Test your changes (check for broken links, formatting issues)
5. Submit a pull request with clear description of changes
6. Wait for review and address any feedback

## What NOT to Contribute

- ❌ Actual exam questions (violates exam policy)
- ❌ Copyrighted material without permission
- ❌ Misleading or inaccurate information
- ❌ Spam or promotional content
- ❌ Personal attacks or inappropriate content

## Questions?

If you have questions about contributing, feel free to:
- Open an issue for discussion
- Reach out to maintainers
- Check existing issues and pull requests

## Recognition

Contributors will be recognized in the project. Thank you for helping make this study guide better for everyone!
