## üí° My Top Tips for Success

### 1. Master the Fundamentals
Don't skip basic networking. Many questions test core concepts:
- TCP vs. UDP
- Port numbers
- Subnetting
- Protocol behavior

### 2. Know Scan Responses Cold
This is heavily tested. Memorize:
- Open TCP port (no firewall): SYN-ACK
- Closed TCP port (no firewall): RST-ACK
- Closed TCP port (firewall): No response or RST-ACK
- Open UDP port: No response
- Closed UDP port (no firewall): ICMP Unreachable

### 3. Practice Windows Commands
You'll see multiple questions on `net` commands:
- `net localgroup` vs. `net group`
- Adding users to groups
- Domain vs. local operations

### 4. Visual Recognition of Permissions
Practice until you can instantly identify:
- SUID: `-rwsr-xr-x` (s in user position)
- SGID: `-rwxr-sr-x` (s in group position)

### 5. Memorize Port Numbers
Know these cold:
- HTTP: 80
- HTTPS: 443
- SSH: 22
- DNS: 53 (TCP for zone transfer, UDP for queries)
- MSSQL: 1433/1434
- RPC: 111
- SNMP: 161/162

### 6. Understand "Why," Not Just "What"
Scenario questions test application, not just memorization:
- Why is SSLv3 unsafe?
- Why use Diffie-Hellman?
- Why is TCP resistant to spoofing?

### 7. Time Management
- Don't spend more than 2 minutes on any question
- Flag difficult questions and move on
- Review flagged questions if time permits
- Trust your first instinct (usually correct)

### 8. Read Questions Carefully
- Look for keywords: "not," "except," "best"
- Eliminate obviously wrong answers
- Choose the BEST answer, not just a correct one

### 9. Stay Calm
- If you don't know an answer, make an educated guess
- Don't panic if you see unfamiliar questions
- Remember: you only need 60% to pass

### 10. Hands-On Practice is Key
- Set up a lab environment
- Practice every command and tool
- Run actual scans and analyze results
- Don't just read‚Äîdo!
